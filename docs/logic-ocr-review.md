## 处理原则

- 判断推理内容以 OCR 后的 Markdown 为主源。
- 与通用行测判断推理体系进行交叉校验。
- 明显属于 OCR 转义、标题编号、公式符号的问题，先纠偏后录入。
- 不把整段例题解析照搬为长文本，优先抽成识别信号、判断规则、步骤和短例题。
- 图形推理暂不放整页图片，先录入通用规律；后续有清晰模块图再补 `image-note`。

## 已纠偏并录入

1. Markdown 中大量标题出现 `1\.`、`2\.` 这类转义。
   - 疑点：OCR/Markdown 转义导致编号异常。
   - 处理：小程序中按正常章节含义拆成知识点，不保留转义符。

2. “既成事实 \+ 原因分析”中的 `\+`。
   - 疑点：加号被转义。
   - 处理：录入为“既成事实 + 原因分析”。

3. 归因论证中“三圈质疑”“四圈支持”只给口诀，没有图示。
   - 疑点：原笔记可能依赖课堂图或图片解释。
   - 处理：用通用表述补正为“同因异果、异因同果、异因异果”，先不补图。

4. 推出关系表中 `\-B→A`、`\-A或B` 等写法。
   - 疑点：负号使用 Markdown 转义。
   - 处理：小程序中改写为“非B→A”“非A或B”等中文化表达。

5. 逻辑公式终极汇总中 `A且B→A→A或B` 写在同一格。
   - 疑点：表格压缩导致层级不清。
   - 处理：未作为独立公式卡录入，只保留推出、矛盾、等价的主干规则。

6. 图形推理章节内容较概括，缺少原图例。
   - 疑点：原笔记可能有图示未进入 Markdown。
   - 处理：按通用图推体系补为“总思路、位置样式、数量属性、空间类”四张知识卡。

## 当前小程序录入范围

- 逻辑论证：归因论证、一般质疑、支持与前提假设、比例类论证、解释说明题
- 推出与分析：充分必要条件、命题真假与逆否、矛盾关系、分析推理、真假推理秒杀
- 定义判断：六要素、排除法
- 类比推理：语义关系、逻辑关系、语法关系
- 图形推理：总思路、位置与样式、数量与属性、空间类图形

## 暂不录入

1. 需要图示支撑的“三圈质疑”“四圈支持”原图。
2. 图形推理中可能存在的原始图例、空间图示。
3. 长篇例题选项逐项解析，只保留能服务方法记忆的短例题。
4. 未与现有训练体系打通的专项训练入口。
