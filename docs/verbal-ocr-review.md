# 言语理解 OCR 复核清单

## 处理原则

- 言语理解内容以 OCR 后的 Markdown 为主源。
- 与通用行测言语理解体系交叉校验后录入。
- 明显属于 Markdown 转义、标题编号、例题遮挡的问题，先纠偏后结构化。
- 例题只保留能服务方法理解的短讲解，不照搬长题干和选项。
- 词语积累不原样塞入完整长表，只保留高频代表项和分类方法。

## 已纠偏并录入

1. 目录中的链接文本存在大量 `\[`、`\]`、`\#`、`\-` 转义。
   - 疑点：Markdown 链接被 OCR/转义保留。
   - 处理：小程序中按真实章节标题拆为知识点，不保留转义符。

2. 章节编号出现 `01\.`、`02\.` 等转义格式。
   - 疑点：编号转义。
   - 处理：录入时统一去掉转义符，按题型组织分类。

3. 部分例题空格被识别为 `\\*\\*\\*\\*` 或混合下划线。
   - 疑点：原文填空横线识别异常。
   - 处理：小程序例题只保留语境和解析方向，不照抄乱码空格。

4. “六大错误选项”实际列出 7 类。
   - 疑点：标题数量与正文不一致。
   - 处理：小程序中按 7 类完整录入。

5. 常考实词表含 `...` 省略段，无法还原全部 180 条。
   - 疑点：Markdown 只展示部分词表。
   - 处理：只录入代表性高频词对，后续可按完整词表扩充。

6. 成语积累表内容较适合速查，不适合作为长阅读卡。
   - 处理：拆成“望文生义”“敬谦误用”“特定对象”三张速查卡。

## 当前小程序录入范围

- 中心理解：结构分析、关键信息、选项分析、标题拟定、下文推断
- 语句表达：语句填入、语句排序
- 细节判断：选项验证和常见偷换
- 逻辑填空：词语辨析、逻辑对应、宏观把握
- 词语积累：常考实词、习惯搭配、易错成语

## 暂不录入

1. 长篇例题原题和完整选项。
2. 常考实词完整 180 条长表。
3. 未完整展示的省略词表项。
4. 未与现有训练体系打通的专项训练入口。

## 后续建议

1. 若后续有完整词表，可单独做“词语速查”数据源，避免详情卡过长。
2. 若新增言语训练，可优先为中心理解、语句排序、逻辑填空绑定 `relatedTraining`。
3. 片段阅读可继续补更多“错误选项识别”短例题。
